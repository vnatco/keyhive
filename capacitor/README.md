# KeyHive Mobile App

Mobile application for [KeyHive](https://keyhive.app) using Capacitor. Wraps the web frontend in a native WebView with biometric unlock, Keychain/Keystore storage, and offline support.

## Files

| File | Purpose |
|------|---------|
| `capacitor.config.json` | Capacitor project config -app ID, web directory, scheme settings, plugin config |
| `package.json` | Dependencies -Capacitor core, iOS platform, biometric auth, secure storage |
| `www/` | Web frontend -generated by the root `build.js`, do not edit directly |
| `ios/` | Xcode project -generated once by `npx cap add ios` (Mac only) |
| `android/` | Android Studio project -generated once by `npx cap add android` |

## How It Works

Capacitor wraps the web app in a native shell using the platform's WebView (WKWebView on iOS, Android WebView on Android). The frontend files are served locally from `www/` via the `capacitor://localhost` scheme.

On launch, Capacitor loads `www/index.html` inside the WebView. The web app detects the Capacitor environment and activates mobile-specific behavior:

- **Biometric unlock** -Face ID / Touch ID / Android fingerprint via Keychain (iOS) and Keystore (Android)
- **Secure storage** -master password stored in the OS secure enclave, never in plain storage
- **Offline mode** -encrypted vault cached in IndexedDB, pending changes synced when back online
- **Platform detection** -`Platform.isMobile()` enables mobile-specific UI and behavior

### Security

- All encryption happens on-device -Argon2id key derivation and AES-256-GCM run inside the WebView via Web Workers
- Content Security Policy injected via meta tag restricting sources to `capacitor://localhost` and the API
- The server only receives encrypted blobs -zero-knowledge architecture preserved on mobile
- Biometric credentials stored in iOS Keychain / Android Keystore (hardware-backed)

## Getting Started

### 1. Build the frontend

From the root of the repository, run:

```bash
node build.js capacitor
```

This copies the contents of `web/` into the `capacitor/www/` folder and patches paths for local loading.

### 2. Install dependencies

```bash
npm install
```

### 3. Add native platforms (one-time setup)

```bash
npx cap add ios        # generates the ios/ Xcode project (Mac only)
npx cap add android    # generates the android/ Android Studio project
```

The `ios/` and `android/` folders are permanent -you never regenerate them.

### 4. Sync and run

#### iOS

```bash
npx cap sync ios       # copies www/ into the Xcode project
npx cap open ios       # opens Xcode -sign and hit Run
```

Requires a Mac with Xcode installed.

#### Android

```bash
npx cap sync android   # copies www/ into the Android Studio project
npx cap open android   # opens Android Studio -build and run
```

Android Studio runs on Windows, Mac, and Linux.
